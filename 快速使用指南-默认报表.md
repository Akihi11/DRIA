# 🚀 快速使用指南 - 默认报表功能

## 立即开始

系统现在会自动提供一个示例报表，您无需任何配置即可：

### 1️⃣ 启动系统

```bash
# 启动后端
python backend/start_server.py

# 启动前端 (新终端)
cd frontend
npm run dev
```

### 2️⃣ 访问报表管理

打开浏览器访问: **http://localhost:5173**

点击导航栏的 **"报表管理"**

### 3️⃣ 查看示例报表

您会看到一个默认的示例报表：

- 📊 报表 ID: `00000000-0000-0000-0000-000000000001`
- 📁 文件大小: ~8KB
- 🔢 包含 5 个示例通道
- 🏷️ 类型标记: `SAMPLE`

### 4️⃣ 下载测试

点击 **"下载"** 按钮，您会得到一个 Excel 文件，包含：

- **工作表 1**: 稳定状态参数汇总
- **工作表 2**: 功能计算结果
- **工作表 3**: 报表信息和使用说明

---

## 生成您自己的报表

示例报表只是开始！要生成真实的数据报表：

### 步骤 1: 准备数据

- 使用 CSV 或 Excel 格式
- 或者使用项目提供的 `samples/Simulated_Data.csv`

### 步骤 2: AI 对话

1. 点击 **"AI 对话"** 页面
2. 上传您的数据文件
3. 与 AI 对话配置报表参数，例如：
   ```
   您: 你好，我想生成一个报表
   AI: 好的，我需要了解一些信息...
   您: 生成稳定状态参数汇总表
   AI: 请告诉我需要展示哪些通道...
   您: 显示所有通道
   AI: 好的，还需要其他配置吗？
   您: 现在生成报表
   ```

### 步骤 3: 查看和下载

1. AI 生成报表后会返回下载链接
2. 或者返回 **"报表管理"** 页面
3. 您会看到新生成的报表（标记为 `REAL`）
4. 点击下载获取您的专属报表

## 验证功能

### 运行诊断脚本

```bash
python test_default_report.py
```

**预期结果**: 所有检查项显示 `[OK]`

### 如果遇到问题

1. **报表列表为空？**

   - 确保后端已重启（以加载默认报表）
   - 运行: `python backend/utils/create_sample_report.py`

2. **下载失败？**

   - 检查 `backend/reports/api_generated/` 目录是否存在
   - 检查是否有 `report_00000000-0000-0000-0000-000000000001.xlsx` 文件

3. **其他问题？**
   - 运行: `python test_report_system.py` 进行全面诊断
   - 查看后端日志输出

## 重要提示

✅ **示例报表是自动的**: 无需手动创建，系统启动时自动加载  
✅ **不会被覆盖**: 您生成的真实报表不会替换示例报表  
✅ **可以删除**: 如果不需要，可以从报表管理页面删除  
✅ **会自动恢复**: 服务重启后，示例报表会重新出现
